About SecLog

Project Overview

SecLog is a comprehensive security check-in/check-out register system designed to streamline visitor management and tracking. The system is ideal for organizations, offices, and facilities that require a reliable method to monitor and record visitor movements.

Key Features

  • Login/Sign-Up: Secure login and sign-up functionality for different user roles (admin, guard, visitor).
  • Home Page: A centralized dashboard with navigation to various functions, including visitor registration and check-in/out processes.
  • Visitor Registration: Register regular and irregular visitors with necessary details.
  • Check-In/Check-Out: Manage the check-in and check-out process for regular and irregular visitors.
  • Dashboard: View all entries with advanced search, filtering options, and print functionality.
  • Settings: Manage user accounts, edit visitor information, and delete entries (admin only).

Technical Details

SecLog is built using a combination of modern web technologies to ensure a smooth and responsive user experience:

  • HTML, CSS, and JavaScript: For structuring and styling the web pages.
  • Bootstrap: A front-end framework to provide a responsive design and modern UI elements.
  • PHP: Server-side scripting for handling user authentication, database interactions, and form processing.
  • MySQL: A relational database management system for storing user data, visitor records, and logs.
  • Apache Server: The web server environment used to host the application.

Project Structure

The project is organized into the following main components:

  • assets/: Contains CSS, JavaScript, and image files used in the project.
  • includes/: Contains PHP files for header, footer, and configuration.
  • pages/: Contains PHP files for various functional pages like login, registration, and dashboard.
  • sql/: Contains SQL scripts for setting up the database schema.
  • index.php: The entry point of the application, typically the home or welcome page.

How to Use

To get started with SecLog:

  1. Set up the database using the provided SQL schema file.
  2. Configure the database connection settings in the config.php file.
  3. Access the application via localhost/seclog and follow the on-screen instructions to log in, register users, and manage visitors.
